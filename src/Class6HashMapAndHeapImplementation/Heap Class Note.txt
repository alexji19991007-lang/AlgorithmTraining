What is heap:
堆（英语：min/max - heap) 亦被称为：优先队列（英语：priority queue）
⽤途：维护⼀个变化的数据集的最优值

Example:
                1
              /   \
             3     2
           /  \   /  \
          5    4 7   null

index: 0   1   2   3   4   5
value: 1   3   2   5   4   7

==========================================================================================================================

性质: 堆的实现通过构造⼆叉树,这种数据结构具有以下性质
1. 堆总是⼀棵完全⼆叉树。complete binary tree
2. 任意节点⼩于（等于）它的所有后裔(descendent)（堆序性）。

Implemented as an unsorted array (VERY IMPORTANT!!!!)
1. index of lChild = myIndex * 2 + 1
2. index of rChild = myIndex * 2 + 2
3. index of parent = (myIndex - 1) / 2
堆的核心任务就是在插入、删除或更新后，把“乱掉”的地方恢复成最小堆。

我们需要了解两个基本的恢复heap性质的操作：percolateUp & percolateDown:
1. 🧹 percolateUp —— 往上冒泡（修复向上违反的堆性质）
    - 什么时候需要 percolateUp？
        ✔️ 新插入元素（在堆尾）
        ✔️ 某个节点的值变小了（可能要往上挪）
    - 直觉：如果一个节点比它的父节点小，那就违反最小堆性质，需要往上换，直到正确位置。
    - 打个比喻：像往水面冒泡——小的元素会一直往上浮。
    - 过程：
        (a) 取当前 index 的父节点 index。
        (b) 如果当前值 < 父节点 → 交换。
        (c) 然后继续检查新的位置，直到满足父节点 ≤ 当前或到达根节点。
2. 🪨 percolateDown —— 往下沉（修复向下违反的堆性质）
    - 什么时候需要 percolateDown？
        ✔️ 删除堆顶（poll）：把最后一个元素放到堆顶
        ✔️ 某个节点的值变大（需要往下放）
    - 直觉：一个节点如果比它的子节点大，就违反最小堆性质，需要往下沉，把更小的孩子换上来。
    - 打个比喻：像石头往水底沉——大的元素会一直向下沉。
    - 过程：
        (a) 找到左右子节点中较小的一个。
        (b) 如果当前值 > 较小孩子 → 交换。
        (c) 向下继续检查，直到不违反性质或者到达叶子。
3. 🌟 为什么这些方法可以恢复最小堆性质？
    - 最小堆只可能在两种地方破坏性质：
        ✔️ 插入元素后 → 只会影响“往上”的路径
        ✔️ 删除堆顶或更新后 → 只会影响“往下”的路径
    - 利用堆是完全二叉树的结构性质：
        往上（父链）最多 log(n) 步
        往下（子链）最多 log(n) 步
    所以，使用简单的交换就能恢复整个堆。
