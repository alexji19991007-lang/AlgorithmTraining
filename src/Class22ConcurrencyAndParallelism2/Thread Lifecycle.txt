==============================
Thread Lifecycle（线程生命周期）
==============================

线程生命周期（Thread Lifecycle）指的是：
一个线程从创建，到执行，再到结束，整个过程中会经历的不同状态。

Java 中线程状态由：
java.lang.Thread.State
这个 enum 定义。

一共有 6 种状态：

1. NEW
2. RUNNABLE
3. BLOCKED
4. WAITING
5. TIMED_WAITING
6. TERMINATED


====================================================
1. NEW（新建状态）
====================================================

线程对象已经创建，
但还没有调用 start()。

示例：

Thread t = new Thread(() -> {
    System.out.println("hello");
});

此时：

t.getState() == NEW

注意：
仅仅 new Thread() 并不会真正启动线程。


----------------------------------------------------
什么时候进入 NEW？
----------------------------------------------------

Thread t = new Thread(...);

创建后立刻进入 NEW。


----------------------------------------------------
什么时候离开 NEW？
----------------------------------------------------

调用：

t.start();

之后进入 RUNNABLE。


====================================================
2. RUNNABLE（可运行状态）
====================================================

这是最容易误解的状态。

RUNNABLE 不代表：
“线程一定正在运行”

而是：

“线程已经准备好运行，等待 CPU 调度，或者正在运行。”

Java 把：

READY（就绪）
RUNNING（运行中）

统一都叫：

RUNNABLE


----------------------------------------------------
什么时候进入 RUNNABLE？
----------------------------------------------------

1.
调用 start()

2.
从 sleep() 醒来

3.
获得 synchronized 锁

4.
join/wait 结束

5.
线程被 scheduler 再次调度


----------------------------------------------------
示例
----------------------------------------------------

Thread t = new Thread(() -> {
    while (true) {
    }
});

t.start();

线程会一直处于：

RUNNABLE


----------------------------------------------------
注意
----------------------------------------------------

即使线程没抢到 CPU，
状态仍然可能是 RUNNABLE。

因为 JVM 不区分：

“正在执行”
和
“等待 CPU”


====================================================
3. BLOCKED（阻塞状态）
====================================================

BLOCKED 专门用于：

等待 synchronized 锁。


----------------------------------------------------
典型场景
----------------------------------------------------

多个线程竞争 synchronized。

例如：

Object lock = new Object();

Thread t1 = new Thread(() -> {
    synchronized (lock) {
        try {
            Thread.sleep(5000);
        } catch (Exception e) {
        }
    }
});

Thread t2 = new Thread(() -> {
    synchronized (lock) {
        System.out.println("got lock");
    }
});

t1.start();
t2.start();

这里：

t1 拿到 lock 后睡 5 秒。

t2 想进入 synchronized(lock)，
但锁已经被占用。

于是：

t2 -> BLOCKED


----------------------------------------------------
BLOCKED 的本质
----------------------------------------------------

等待 monitor lock（监视器锁）。


----------------------------------------------------
注意
----------------------------------------------------

BLOCKED != WAITING

BLOCKED：
    等 synchronized 锁

WAITING：
    等别人通知


====================================================
4. WAITING（无限等待）
====================================================

线程进入等待状态，
直到别的线程主动唤醒它。

没有时间限制。


----------------------------------------------------
会导致 WAITING 的方法
----------------------------------------------------

1.
Object.wait()

2.
Thread.join()

3.
LockSupport.park()


----------------------------------------------------
例子：join()
----------------------------------------------------

Thread t1 = new Thread(() -> {
    try {
        Thread.sleep(3000);
    } catch (Exception e) {
    }
});

Thread t2 = new Thread(() -> {
    try {
        t1.join(); // 当前线程等待 t1 执行结束
    } catch (Exception e) {
    }
});

t1.start();
t2.start();

这里：

t2 等待 t1 执行结束。

因此：

t2 -> WAITING


----------------------------------------------------
wait() 示例
----------------------------------------------------

Object lock = new Object();

synchronized(lock) {
    lock.wait();
}

线程进入 WAITING，
直到别人：

lock.notify();

或者：

lock.notifyAll();


----------------------------------------------------
WAITING 的特点
----------------------------------------------------

无限期等待。

必须依赖：

1. notify()
2. notifyAll()
3. interrupt()
4. 被等待线程结束


====================================================
5. TIMED_WAITING（限时等待）
====================================================

和 WAITING 很像，
但有时间限制。


----------------------------------------------------
会导致 TIMED_WAITING 的方法
----------------------------------------------------

1.
Thread.sleep()

2.
wait(timeout)

3.
join(timeout)

4.
parkNanos()

5.
parkUntil()


----------------------------------------------------
sleep() 示例
----------------------------------------------------

Thread.sleep(3000);

线程会进入：

TIMED_WAITING

3 秒后自动恢复。


----------------------------------------------------
join(timeout)
----------------------------------------------------

t.join(5000);

最多等 5 秒。

超时后自动恢复。


----------------------------------------------------
TIMED_WAITING vs WAITING
----------------------------------------------------

WAITING：
    无限等待

TIMED_WAITING：
    有时间限制


====================================================
6. TERMINATED（终止状态）
====================================================

线程执行完毕后进入 TERMINATED。


----------------------------------------------------
示例
----------------------------------------------------

Thread t = new Thread(() -> {
    System.out.println("done");
});

t.start();

线程 run() 执行结束后：

t.getState() == TERMINATED


----------------------------------------------------
注意
----------------------------------------------------

线程结束后：

不能再次 start()

否则：

IllegalThreadStateException


====================================================
完整生命周期流程
====================================================

NEW
  ↓ start()

RUNNABLE
  ↓ waiting for lock
BLOCKED
  ↓ lock acquired
RUNNABLE
  ↓ wait/join
WAITING
  ↓ notified
RUNNABLE
  ↓ sleep(timeout)
TIMED_WAITING
  ↓ timeout
RUNNABLE
  ↓ run() finished
TERMINATED


====================================================
常见方法对应状态总结
====================================================

Thread.sleep()
    -> TIMED_WAITING

Object.wait()
    -> WAITING

Object.wait(timeout)
    -> TIMED_WAITING

Thread.join()
    -> WAITING

Thread.join(timeout)
    -> TIMED_WAITING

waiting for synchronized
    -> BLOCKED

run() completed
    -> TERMINATED


====================================================
一个非常重要的区别
====================================================

BLOCKED：
    等锁（synchronized）

WAITING：
    等通知（notify/join）

TIMED_WAITING：
    等时间结束


====================================================
面试高频问题
====================================================

Q:
sleep() 会释放锁吗？

A:
不会。

sleep() 期间仍然持有 synchronized 锁。


----------------------------------------------------

Q:
wait() 会释放锁吗？

A:
会。

wait() 会释放 monitor，
否则别人无法 notify。


----------------------------------------------------

Q:
为什么 wait() 必须在 synchronized 中调用？

A:
因为 wait/notify 操作的是 monitor lock，
必须先持有 monitor。


----------------------------------------------------

Q:
BLOCKED 和 WAITING 最大区别？

A:

BLOCKED：
    等 synchronized 锁

WAITING：
    等线程间通信

====================================================

学习建议
====================================================

真正理解线程生命周期，
最重要的是理解：

1.
线程到底在“等什么”

2.
wait/sleep/join 的区别

3.
synchronized 和 monitor 的关系

4.
CPU 调度 vs JVM 状态

5.
线程通信机制


====================================================
一句话总结
====================================================

NEW
    线程刚创建

RUNNABLE
    可以运行/正在运行

BLOCKED
    等 synchronized 锁

WAITING
    无限等待别人唤醒

TIMED_WAITING
    限时等待

TERMINATED
    线程结束
